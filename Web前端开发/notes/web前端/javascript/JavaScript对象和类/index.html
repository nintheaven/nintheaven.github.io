JavaScript对象和类

: Web前端开发
发布于 : 2022-10-11

对象的基本用法

创建对象

方式1：对象字面量

声明空对象

var person = {};

对象中可以有属性和方法

var person = {
    firstName: "君鹏",
    lastName: "贾",
    age: 18,
    greeting: function() {
        alert(this.lastName + this.firstName + "，你妈妈喊你回家吃饭!");
    }
};

对象中属性值可以是另一个对象

var person = {
    name: {
        firstName: "君鹏",
        lastName: "贾"
    },
    age: 18,
    greeting: function() {
        alert(this.lastName + this.firstName + "，你妈妈喊你回家吃饭!");
    }
};

方式2：封装一个创建对象的函数(在函数中创建对象，然后返回对象)

function createNewPerson(name) {
    var obj = {};
    obj.name = name;
    obj.greeting = function() {
        alert('Hi! I\'m ' + this.name + '.');
    }
    return obj;
}

方式3：构建函数

function Person(name) {
    this.name = name;
    this.greeting = function() {
        alert('Hi! I\'m ' + this.name + '.');
    };
}

其它方式

new Object()

var person = new Object();
person.name = 'Chris';
person['age'] = 38;
person.greeting = function() {
    alert('Hi! I\'m ' + this.name + '.');
}

或者

var person = new Object({
    name: 'Chris',
    age: 38,
    greeting: function() {
        alert('Hi! I\'m ' + this.name + '.');
    }
});

ES5中，可以使用Object.create()

var person2 = Object.create(person1);

使用对象

访问对象成员

点表示法

person.name.firstName;
person.age;
person.greeting();

方括号表示法

person["name"]["firstName"];
person["age"]
person["greeting"]()

设置对象成员的值

使用等号赋值

person.age=20;

对象原型

通过原型这种机制，JavaScript 中的对象从其他对象继承功能特性；这种继承机制与经典的面向对象编程语言的继承机制不同。

function Person() {}
var person = new Person();

获取原型对象

console.log(Person.prototype);
console.log(Object.getPrototypeOf(person));

修改原型

Person.prototype.name = '';
Person.prototype.greeting = function() {
    alert("Hi！" + this.name);
};

原型链

每个对象或者函数(本质上也是对象)，均有自己的原型对象；

每个实例对象（object）都有一个私有属性（称之为__proto__）指向它的构造函数的原型对象（prototype ）。
该原型对象也有一个自己的原型对象(__proto__)，层层向上直到一个对象的原型对象为 null。
根据定义，null 没有原型，并作为这个原型链中的最后一个环节。

原型继承

prototype 属性的值是一个对象，我们希望被原型链下游的对象继承的属性和方法，都被储存在其中。

属性查找：上溯原型链

看2个例子

person.name;
  • 浏览器首先检查，person对象是否有name属性；
  • 如果没有，则浏览器检查 person 对象的原型对象是否具有name属性；
  • 这里有
person.toString();
  • 浏览器首先检查，person 对象是否具有可用的 toString() 方法；
  • 如果没有，则浏览器检查 person 对象的原型对象是否具有可用的 toString() 方法；
  • 如果也没有，则浏览器检查 person 对象原型对象的原型对象(即Object的原型对象)，是否具有可用的 toString() 方法。
  • 这里有这个方法，于是该方法被调用。

最佳实践

一种极其常见的对象定义模式是，在构造器（函数体）中定义属性、在 prototype 属性上定义方法。


function Test(a,b,c,d) {
  // 属性定义
};
Test.prototype.x = function () { ... }
Test.prototype.y = function () { ... }

通过原型实现继承

定义Person类

function Person(first, last, age, gender, interests) {
    this.name = {
        first,
        last
    };
    this.age = age;
    this.gender = gender;
    this.interests = interests;
};

Person.prototype.greeting = function() {
    alert('Hi! I\'m ' + this.name.first + '.');
};

定义Teacher类，继承Person

function Teacher(first, last, age, gender, interests, subject) {
    // 运行Person构造函数
    Person.call(this, first, last, age, gender, interests);

    this.subject = subject;
}

// Teacher.prototype现在是空，让其继承Person.prototype的所有属性和方法
Teacher.prototype = Object.create(Person.prototype);
// Teacher()的prototype的constructor属性现在指向的是Person(),让其指向Teacher()
Teacher.prototype.constructor = Teacher;

this

this指当前对象-访问属性或调用方法的对象。

理解了原型链和原型继承，this的查找顺序就很好理解了。

var person = {
    firstName: "君鹏",
    lastName: "贾",
    age: 18,
    greeting: function() {
        alert(this.lastName + this.firstName + "，你妈妈喊你回家吃饭!");
    }
};

person.greeting()中,this指的就是person.

ES6中的类与继承

ECMAScript6 引入了一套新的关键字用来实现 class。
使用基于类语言的开发人员会对这些结构感到熟悉，但它们是不同的。JavaScript仍然基于原型，class只是语法糖而已。
这些新的关键字包括 class, constructor，static，extends 和 super。

class Polygon {
    constructor(height, width) {
        this.height = height;
        this.width = width;
    }
}

class Square extends Polygon {
    constructor(sideLength) {
        super(sideLength, sideLength);
        this.name = "Square";
    }

    set sideLength(newLength) {
        this.height = newLength;
        this.width = newLength;
    }

    get area() {
        return this.width * this.height;
    }
}
