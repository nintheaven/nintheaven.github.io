模板方法模式介绍

在父类中定义处理流程、在子类中实现具体逻辑的模式，称为模板方法模式。

一套模板（抽象父类），根据子类的不同实现，表现出不同的行为，这就是使用模板方法模式的理由。

使用模板方法模式可使逻辑处理通用化，父类中的模板方法定义了通用的算法框架（一系列步骤），子类只负责实现算法框架中特定的某些步骤。

站在父类的角度思考问题

父类中的方法可以调用自己的（抽象）方法，实现通用的逻辑。要求子类实现抽象方法的具体的逻辑。
