javac编译报错：

error: unmappable character for encoding ASCII

原因及解决

这个报错是因为源码中有中文字符，但是计算机不支持该字符编码导致的。

因此，解决这个报错的途径就有2个：要么删除掉源码中的中文字符，要么让计算机支持中文字符。

让计算机支持中文字符，需要添加环境变量(例如 /etc/profile文件)，在文件末尾加上

export JAVA_TOOL_OPTIONS=-Dfile.encoding=UTF8

javac编译报错

class file has wrong version 55.0, should be 52.0

报错原因及解决

这个报错是因为用到的第三方库编译时使用的java版本高于自己当前的java版本。

因此，解决这个报错的途径有2个：要么使用第三方库较低的编译版本，要么自己使用较高的java版本进行编译

Java版本的主版本号如下：

J2SE 8 = 52

J2SE 7 = 51

J2SE 6.0 = 50

J2SE 5.0 = 49

JDK 1.4 = 48

JDK 1.3 = 47

JDK 1.2 = 46

JDK 1.1 = 45

jar xvf xxx.war

修改war包中的文件内容

先解压，然后修改，最后将修改更新到war包中

jar uvf xxx.war 修改的文件
