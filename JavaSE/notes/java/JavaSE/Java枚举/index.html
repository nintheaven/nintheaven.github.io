Java枚举

: JavaSE
发布于 : 2022-10-11

什么是枚举？

定义的一个只包含有限个值得集合。

为什么使用枚举

举个例子，服装的尺寸一般只有小,中,大,超大4种。

要在代码中表示这4种尺寸，菜鸟程序员可能会定义常量1,2,3,4来实现

public class TailorShop
{
    public static final int SMALL=1;
    public static final int MEDIUM=2;
    public static final int LARGE=3;
    public static final int EXTRA_LARGE=4;
}

或者定义分别用字符S,M,L,XL表示

public class TailorShop
{
    public static final String SMALL="S";
    public static final String MEDIUM="M";
    public static final String LARGE="L";
    public static final String EXTRA_LARGE="XL";
}

然后在使用的时候，将一个数字，或者字符与这些常量进行比较

public static void makeClothes(int size)
{
    if(size==SMALL)
    {
        System.out.println("做小号衣服");
    }
    if(size==MEDIUM)
    {
        System.out.println("做中号衣服");
    }
    //...
}

显然，无论是使用常量数字还是字符串，都有可能在其中保存了一个错误的值，导致难以发觉的bug。
而且，用几个常量表示“尺寸”这个概念，也不符合面相对象思想。

而枚举，就没有上面的问题。

Java中如何使用枚举

枚举的用法很简单，就是一个简化版的类，将定义类时的class关键字换为enum，然后在其中声明枚举项即可。

enum Size
{
    SMALL,MEDIUM,LARGE,EXTRA_LARGE;
}

然后，这样使用

public static void makeClothes(Size size)
{
    if(size==Size.SMALL)
    {
        System.out.println("做小号衣服");
    }
    if(size==Size.MEDIUM)
    {
        System.out.println("做中号衣服");
    }
    //...
}

枚举类型可以用在switch语句中

public static void makeClothes(Size size)
{
    switch(size)
    {
        case SMALL:break;
        case MEDIUM:break;
        case LARGE:break;
        case EXTRA_LARGE:break;
    }
}

需要注意的是，case块后直接使用枚举项的名称即可，不需要（也不允许）使用Size.SMALL这样的写法。

枚举的其它知识

在枚举中声明的枚举项，都是枚举类型的实例。

如果需要，枚举中也可以声明字段，方法和构造器

public enum Size
{
    SMALL("S"),MEDIUM("M"),LARGE("L"),EXTRA_LARGE("XL");
    
    private String abbreviation;
    
    Size(String abbreviation)
    {
        this.abbreviation=abbreviation;
    }
    
    public String getAbbreviation()
    {
        return this.abbreviation;
    }
}

注意：枚举中的构造器默认(必须)是private的，也就是说，枚举类型只能在内部声明实例，而不能在外部创建实例。

Java中自定义的枚举类型，实际上会被编译器转换为普通的Java类，它必定是java.lang包下的Enum<T>类的子类。

例如，上面定义的枚举类型Size，实际继承自Enum<Size>。

由于任何自定义枚举类型都是Enum类型的子类，所以，任何枚举类型都从Enum类型继承了一些字段和方法

  • String name(),获取枚举项的名字，例如 SMALL

  • int ordinal(),获取枚举项的序数，序数从0开始，例如SMALL是0，MEDIUM是1

  • String toString(),Enum重写了该方法，返回name字段的值，将Enum转换为String

    String small=Size.SMALL.toString()

  • static <T extends Enum<T>> T valueOf(Class<T> enumType,String name),将String换为Enum

    Size SMALL=Enum.valueOf(Size.class,"SMALL");

  • values()静态方法，返回包含所有枚举项的数组。
    这是一个有趣的方法，它既不是我们自己定义的，父类Enum中也没有这个方法，那它是从哪来的呢？
    原来，这是Java编译器在编译时为我们添加的。

    反编译枚类的class文件

    javap Size.class

    得到如下结果

    public final class enumeration.Size extends java.lang.Enum<enumeration.Size> {
public static final enumeration.Size SMALL;
public static final enumeration.Size MEDIUM;
public static final enumeration.Size LARGE;
public static final enumeration.Size EXTRA_LARGE;
public static enumeration.Size[] values();
public static enumeration.Size valueOf(java.lang.String);
public java.lang.String getAbbreviation();
static {};
}

    可以看到，Size[] values()方法已经添加到Size中了。
