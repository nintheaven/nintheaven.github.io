JEP409：密封类

密封类或接口在定义时，就显式的限制哪些类或接口可以继承或实现它，规则如下：

  • 使用 sealed 关键字声明类或接口
  • 使用 permits 关键字声明将要继承该基类或接口的派生类或子接口
  • 要继承被 sealed 关键字修饰的类或接口，子类或子接口必须使用 sealed 或 non-sealed 关键字修饰。如果派生类或子接口使用 sealed 修饰，其本身又成为了密封类或密封接口。

示例1

密封类及其子类

/* 声明Animal为密封类，只允许Cat和Dog类继承它 */
public sealed class Animal permits Dog,Cat {
}

/* 声明Cat不是密封类 */
public non-sealed class Cat extends Animal{
}

public non-sealed class Dog extends Animal{
}

密封接口及其子接口或实现类

/* 声明IAction为密封接口，只允许IJump接口继承它 */
public sealed interface IAction permits IJump {
}

public non-sealed interface IJump extends IAction {
}

/* 因为IJump是非密封接口，所以不受限制，任何类都可以实现它 */
public non-sealed class Dog extends Animal implements IJump{
}
