git分支

2022-10-11

分支基础操作

本地分支

查看分支

git branch

创建分支

git branch test

切换分支

git checkout test

创建并切换分支

git checkout -b test

合并另一个分支到当前分支

git merge test

将test分支中的 res 文件夹合并到当前分支

git checkout test -- res

合并冲突

在两个不同的分支中，对同一个文件的同一个部分进行了不同的修改，Git 就没法干净的合并它们。

发生合并冲突后，需要修改发生冲突的文件，然后使用 git add 命令标记其冲突已解决。

删除分支

git branch -d test

远程分支

添加远程分支

git remote add origin git@github.com:lypzzzzz/blog.git

跟踪远程分支

git branch --set-upstream-to=origin/main main

添加子模块

git submodule add git@github.com:next-theme/hexo-theme-next.git themes/next

如果报错

fatal: 'themes/next' already exists in the index

执行以下命令

git rm --cached themes/next

如果报错

error: the following file has staged content different from both the
file and the HEAD:
    themes/next
(use -f to force removal)

按照提示，添加 -f 选项

git rm --cached -f themes/next

然后重新运行添加子模块的命令即可。
