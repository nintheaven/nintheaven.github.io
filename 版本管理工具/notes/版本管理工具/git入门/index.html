git入门

版本管理工具 2022-10-11

config, init, clone, add, commit, restore, checkout, log, .ignore文件, remote, push, pull, fetch, 标签，命令别名

基础概念

3个目录

  • 工作目录
  • 暂存区
  • 仓库目录

3个状态

  • 已修改
  • 已暂存：git add
  • 已提交: git commit

配置

3个配置文件

  • /etc/gitconfig，全局配置，使用 git config --system
  • ~/.gitconfig 或 ~/.config/git/config，当前用户配置， 使用git config --global
  • .git/config，当前仓库配置，使用命令 git config --local

配置用户信息

git config --global user.name "John Doe"

git config --global user.email johndoe@example.com

配置编辑器

git config --global core.editor vim

查看配置信息

git config --list

git config user.name

基本操作

获取仓库

初始化仓库

git init

克隆仓库

git clone <仓库地址>

克隆仓库并重命名

git clone <仓库地址> <新名称>

查看信息

查看文件状态

git status

git status -s # –short

查看已修改但尚未暂存的变化内容

git diff

查看已暂存的将要添加到下次提交的变化内容

git diff --staged

git diff --cached

查看提交历史

git log

git log --graph # 显示Author与Date

git log --pretty=oneline

暂存,取消暂存,提交,移除

跟踪(暂存)文件或目录（递归的跟踪该目录下的所有文件）

git add <文件或目录名称列表>

git add *

git add --all

提交文件

git commit

git commit -m "提交信息"

暂存并提交(不包含新建的文件)

git commit -a -m "提交信息"

重新提交最近一次提交

git commit --amend

取消暂存(从暂存区移除)

git reset HEAD -- <file>...

git restore --staged <file>...

取消修改，将其还原为上次提交时的样子

git checkout -- <file>...

移除文件

  • 从暂存区域和工作目录同时移除

    git rm <文件列表>

    git rm -f <文件列表>

  • 从git仓库和暂存区域移除，但在工作目录中保存

    git rm --cached <文件列表>

    git rm -r --cached <目录>

移动（重命名）文件

git mv xxx yyy

忽略文件

创建 .gitignore 文件，用于定义忽略规则：

  • 用#注释
  • 匹配模式以 / 开头，避免递归
  • 匹配模式以 / 结尾，指定目录
  • 在匹配模式前面加 ! 取反
  • 支持标准glob模式
    • *匹配0个或多个任意字符
    • ?匹配1个任意字符
    • [a-z]匹配任意1个在其中的字符
    • a/**/z匹配任意中间目录

示例

远程仓库

查看远程仓库

git remote

git remote -v

git remote show <remote_name>

添加远程仓库

git remote add <remote_name> <URL>

重命名远程仓库的简称

git remote rename <old_name> <new_name>

移除一个远程分支

git remote rm <remote_name>

拉取远程仓库中的内容(不会自动合并)

git fetch <remote_name>

clone一个仓库,会自动将其添加为远程仓库，用origin作为其简称。

拉取远程仓库中的内容（如果设置当前分支跟踪一个远程分支，pull命令自动拉取并合并远程分支到当前分支）

git pull

git pull <remote_name>

git pull <remote_name> <remote_branch>

git pull origin main

推送到远程仓库

git push

git push -u <remote_name>

git push -u <remote_name> <remote_branch>

git push -u origin main

推送到远程仓库，并指定上游仓库分支

git push --set-upstream origin main

标签

查看

查看全部标签

git tag

git tag --list

查看标签(使用通配符)

git tag --list "v1.0.*"

查看标签具体信息

git show 标签名

创建,删除,推送

给当前提交创建附注标签

git tag -a 标签名 -m "描述信息"

给当前提交创建轻量标签

git tag 标签名

给历史提交打标签

可以使用提交校验和信息，给历史提交打标签(使用git log查看校验和)

git tag -a 标签名 (部分)校验和

推送标签到远程仓库

使用git push时，tag信息不会被推送到远程仓库，需要使用以下命令进行推送

git push origin 标签名

git push origin --tags

删除本地标签

git tag -d 标签名

删除远程仓库中的标签

git push origin --delete 标签名

别名

可以为一些git命令设置别名

例如，取消暂存命令如下

git restore --staged <file>...

创建别名

git config --global alias.unstage "restore --staged"

使用别名替代git命令

git unstage <file>...
