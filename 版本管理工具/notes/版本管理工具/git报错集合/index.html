git add 命令长时间无响应，无法add文件

报错原因是windows系统对行结束符号的脑残设计导致的，解决办法如下

1.删除.git目录下的index.lock文件

2.修改.git/conf文件，在core项下添加以下配置

[core]
    autocrlf = false
    safecrlf = false

git push报错：error: remote unpack failed: eof before pack header was fully read

目前只有暴力解决方案：删除本地仓库，重新创建git仓库，推送到remote仓库

1.删除.git目录

2.初始化git仓库

git init

3.添加远程仓库

git remote add origin git@github.com:<username>/<rep_name>.git

4.重新push

git add --all

git commit -m 'commit'

git push

git push报错

E:\Projects\algorithm>git push gitee "main"
error: src refspec main does not match any
error: failed to push some refs to 'gitee.com:lypzzzzz/algorithm.git'

是由于本地分支（master）与远程分支（main）名称不一致造成的，修改本地分支名称

git branch -m master main
