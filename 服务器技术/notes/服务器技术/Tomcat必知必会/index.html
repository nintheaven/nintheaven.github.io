Tomcat服务器安装

首先，下载安装包
下载地址: https://tomcat.apache.org/

然后安装，Tomcat安装包只是一个压缩包文件，只需要解压到安装目录即可。
例如，在linux中，要将其安装到 /opt 目录中,运行命令
sudo tar zxvf apache-tomcat-9.0.36.tar.gz -C /opt/
此时，opt下的tomcat目录属于root用户，root用户组，为了避免权限问题，再运行以下命令
sudo chown -R ubuntu.ubuntu apache-tomcat-9.0.36/
这里的ubuntu是用户名和用户组名

至此，Tomcat已成功安装。

卸载Tomcat服务器很简单，只需要将Tomcat文件夹删除即可(如果配置了环境变量，也要记得删除)。

Tomcat环境配置

在启动服务器之前，需要确保配置了JAVA_HOME 或 JRE_HOME 环境变量，以及 CATALINA_HOME 和 CATALINA_BASE 环境变量

这里没有将 bin/ 目录加到 PATH 环境变量中，因为没必要

# JDK
JAVA_HOME="/opt/jdk/jdk-11.0.12"
PATH=$PATH:$JAVA_HOME/bin

# Tomcat
CATALINA_HOME="/opt/apache-tomcat-9.0.54"
CATALINA_BASE="/opt/apache-tomcat-9.0.54"

export PATH

Tomcat服务的启动与停止

启动服务

http://localhost:8080/

On Windows:

%CATALINA_HOME%\bin\startup.bat
or
%CATALINA_HOME%\bin\catalina.bat start

On *nix:

$CATALINA_HOME/bin/startup.sh
or
$CATALINA_HOME/bin/catalina.sh start

停止服务

On Windows:

%CATALINA_HOME%\bin\shutdown.bat
or
%CATALINA_HOME%\bin\catalina.bat stop

On *nix:

$CATALINA_HOME/bin/shutdown.sh
or
$CATALINA_HOME/bin/catalina.sh stop

部署Web项目有几种不同的方式

1.直接将项目文件夹复制到webapps目录下

2.通过war包的方式，将项目文件打包为war包，复制到webapps目录下。

这种方式的好处是，Tomcat可以在不重启的情况下，自动管理war包

3.在conf/server.xml中的Host标签中配置

<Host>
    <Context docBase="/home/ubuntu/Projects/blog" path="/blog" />
</Host>

4.在conf/Catalina/localhost目录下，为每个项目创建单独的配置文件

文件名为blog.xml，文件名blog就是项目的上下文路径。配置内容如下

<Context docBase="/home/ubuntu/Projects/blog" />

警告和报错

tomcat启动日志中文乱码，修改 conf 目录下的 logging.properties 配置文件，修改控制台编码为GBK

java.util.logging.ConsoleHandler.level = FINE
java.util.logging.ConsoleHandler.formatter = org.apache.juli.OneLineFormatter
#java.util.logging.ConsoleHandler.encoding = UTF-8
java.util.logging.ConsoleHandler.encoding = GBK
