springboot使用actuator

笔记 : SpringBoot系列
发布于 : 2022-10-11

官方文档：https://docs.spring.io/spring-boot/docs/2.5.8/reference/html/actuator.html

Actuator公开一些Rest端点，用于监控SpringBoot应用程序运行时的内部状况。

启用Actuator

添加依赖

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>

启动程序，访问 http://localhost:8080/actuator

{
  "_links": {
    "self": {
      "href": "http://localhost:8080/actuator",
      "templated": false
    },
    "health": {
      "href": "http://localhost:8080/actuator/health",
      "templated": false
    },
    "health-path": {
      "href": "http://localhost:8080/actuator/health/{*path}",
      "templated": true
    }
  }
}

启用端点(Endpoints)

SpringBoot内置了很多有用的端点，也可以添加自定义端点。

常用的内置端点如下，完整端点列表查看Spring文档：

  • health：查看程序的健康指标
  • info：显示应用程序的信息，默认为空Json对象，可以在配置文件中使用 info 自定义
  • configprops：查看通过 @ConfigurationProperties 注解设置的配置属性信息，包括springboot自动配置的和自己手动配置的。
  • env：查看环境相关的属性信息
  • metrics：查看JVM、CPU、服务器等运行环境相关的信息
  • beans：查看由Spring容器管理的全部Bean
  • loggers：查看日志配置信息
  • mappings：查看应用中通过 @RequestMapping 注解的全部控制器方法的相关信息（HTTP请求方式、请求头、URL路径等）
  • shutdown：通过此接口可以优雅的关闭应用，需要使用POST请求。

默认情况下，只有 health 端点是启用的。

关闭全部端点（enabled-by-default默认为true，只启用 health 端点）

management:
  endpoints:
    enabled-by-default: false

启用某个端点

注意，除了要设置 management.endpoint.<endpoint-id>.enabled = true 启用端点以外，还要使用 management.endpoints.web.exposure.include = <endpoint-id> 将其暴露出去，否则端点还是无法访问，简单点说就是 **既要启用(enabled)，又要暴露(exposure)**。

另外，还可以使用 management.endpoints.web.exposure.exclude 排除要暴露的端点。

management:
  endpoints:
    enabled-by-default: false
    web:
      exposure:
        include: beans
  endpoint:
    beans:
      enabled: true

启用多个端点只需要使用添加多个management.endpoint.<endpoint-id>.enabled = true，并在 management.endpoints.web.exposure.include 中暴露出去即可。一般的做法是：先禁用全部端点，再设置要启用的部分端点，然后使用通配符星号 * 将全部端点暴露（include）出去，最后再排除不想暴露的端点（exclude）

management:
  endpoints:
    # 禁用全部端点
    enabled-by-default: false
    web:
      exposure:
        # 暴露全部端点
        include: '*'
        # 排除部分端点
        exclude: env,startup,shutdown
  # 设置要启用的端点      
  endpoint:
    health:
      enabled: true
    info:
      enabled: true
    metrics:
      enabled: true
    configprops:
      enabled: true
    beans:
      enabled: true
    mappings:
      enabled: true
    env:
      enabled: true
    startup:
      enabled: true
    shutdown:
      enabled: true
    loggers:
      enabled: true

自定义端点

设置info信息

info:
  app:
    encoding: "UTF-8"
    java:
      source: "8"
      target: "8"

重启应用，访问 info 端点：http://localhost:8080/actuator/info

{
  "app": {
    "encoding": "UTF-8",
    "java": {
      "source": "8",
      "target": "8"
    }
  }
}
