SpringBoot使用devtools实现开发环境热部署

笔记 : SpringBoot系列
发布于 : 2021-10-21

devtools的原理

devtools中提供了2个ClassLoader，一个Bean类加载器用于加载第三方jar包等不会改变的类，另一个Restart类加载器用于加载正在修改的那些类。

当classpath目录下的文件修改后，devtools自动重启应用，但是只有Restart类加载会运行。所以启动速度会根块。

使用步骤

添加spring-boot-devtools依赖后，自动生效

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-devtools</artifactId>
    <optional>true</optional>
    <scope>runtime</scope>
</dependency>

在idea中，修改文件后，构建应用（Ctrl+F9）,而不必重启应用。

也可以在配置文件中进行配置，例如，禁用自动重启

spring.devtools.restart.enabled: off
