Controller中的常用注解

  • @Controller：声明一个类为控制器类

  • @ResponseBody：用于控制器类或方法上，声明控制器方法的返回值即为HTTP响应体，不再经过视图进行处理。

  • @RestController：相当于同时使用 @Controller 和 @ResponseBody 2个注解

  • @RequestMapping：声明一个方法为控制器方法，用来处理HTTP请求

  • @GetMapping、@PostMapping、@PutMapping、@DeleteMapping、@PatchMapping

  • @PathVariable：用于控制器方法参数，获取捕获的URL路径变量

  • @RequestParam：用于控制器方法参数，获取请求参数

    In Spring MVC, “request parameters” map to query parameters, form data, and parts in multipart requests. This is because the Servlet API combines query parameters and form data into a single map called “parameters”, and that includes automatic parsing of the request body.

  • @RequestBody：用于控制器方法参数，获取请求体

  • @RequestHeader：用于控制器方法参数，获取请求头信息

  • @CookieValue：用于控制器方法参数，获取http请求中的Cookie信息

  • @SessionAttribute：用于控制器方法参数，获取会话属性

  • @RequestAttribute：用于控制器方法参数，获取请求属性

  • @MatrixVariable：用于控制器方法参数，获取请求参数中的矩阵数据。springboot默认禁用矩阵变量。

Controller响应

  • @ResponseBody：使用该注解的控制器方法，其返回值即响应体

  • HttpEntity<B>

  • ResponseEntity<B>：继承HttpEntity<B>，更方便使用

  • String：返回视图名称

  • View，ModelAndView：返回视图

使用静态资源

注意：如果自定义了 WebMvcConfigurer 配置类并启用了webmvc @EnableWebMvc，下面的默认行为无效

springboot默认在下面几个目录查找静态资源：

  • classpath:/META-INF/resources
  • classpath:/resources/
  • classpath:/static/
  • classpath:/public/

修改静态资源默认查找路径，可以使用 spring.web.resources.static-locations 配置属性进行修改。例如，添加一个本地文件系统中的目录

classpath表示类路径，file表示本地文件系统。

在前后端分离项目中，如果无需要后端提供静态资源，也可以禁用静态资源功能

修改静态资源URL前缀，可以使用配置 spring.mvc.static-path-pattern 自定义静态资源的URL前缀,默认为 /**

这样配置后，访问静态资源就必须使用前缀 /static。例如，访问图片 /static/images/a.png， 之前访问URL为 /images/a.png，使用配置后，就成了 /static/images/a.png

注意

如果定义了 WebMvcConfigurer 的实现类，上面的配置属性不会生效，导致访问静态资源404。此时，必须重写默认方法 addResourceHandlers(ResourceHandlerRegistry)

欢迎页面index

注意：如果自定义了 WebMvcConfigurer 配置类并启用了webmvc @EnableWebMvc，下面的默认行为无效

静态资源访问URL前缀必须是默认的 /**

spring-boot支持从配置的静态文件路径和模板路径下查找欢迎页。首先从静态文件路径下查找 index.html，如果没有找到，就去模板路径下查找 index 模板。

如果从以上2个地方都没找到，才交给Controller去动态匹配 ServletContext 路径，上下文路径默认是 */*。

如果使用了模板引擎，推荐将欢迎页放到模板目录中。

HTML表单提交GET和POST之外的HTTP请求

使用HTML表单提交时，默认只支持 GET 和 POST 方式，spring-mvc对其它请求方式的支持使用了Filter技术。

springboot应用默认配置了 HiddenHttpMethodFilter 过滤器，POST请求交给控制器处理之前，会从请求体中获取 _method 隐藏字段，使用装饰者设计模式将其包装到 HttpServletRequest 对象中，进而将POST请求转换为隐藏字段设置的请求方式。

核心源码

这个过滤器默认是关闭的，需要在配置中手动打开它

要想使用HTML表单发送 PUT，PATCH，DELETE 请求方式，需要以 POST 的方式发起，然后，在表单中添加一个 _method 隐藏字段，其值为要使用的HTTP请求方式
