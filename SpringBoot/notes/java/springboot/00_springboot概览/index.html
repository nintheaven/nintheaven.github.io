springboot必须掌握的核心注解

@SpringBootApplication

用在类上，表示该类是一个SpringBoot应用，它是下面3个注解的组合：

  • @SpringBootConfiguration，配置类
  • @EnableAutoConfiguration，启用自动配置
  • @ComponentScan，组件扫描

@Configuration

用在类上，表示该类是一个Spring配置类。在该类中使用@Bean注解声明方法，用于向Ioc容器注入Bean。

这个注解有个叫 proxyBeanMethods 的boolean属性，表示是否启用代理Bean模式。true时，从IoC中获取对象，否则，每次创建一个新的对象。默认开启。

@Bean

用在方法或注解上，向IoC容器中注入Bean实例。

@Component

用在类上，表示该类是一个组件。它有以下几个语义化的注解，其作用是一样的：

  • @Controller
  • @Service
  • @Repository

@RestController

用在类上，是@Controller和@ResponseBody的组合注解。

@RequestMapping

用在类或方法上，用于将web请求映射到指定的方法上。它有以下几个与HTTP请求方法对应的注解：

  • @GetMapping
  • @PostMapping
  • @PutMapping
  • @PatchMapping
  • @DeleteMapping

@Autowired

自动装配，用在构造器、字段、方法、方法参数、注解类型上。

根据类型(byType)在IoC容器中查找Bean，需要结合@Qualifier注解实现根据名称(byName)查找Bean。

@Resource

自动装配，用在类、字段、方法上。

默认byName查找Bean（也可以设置name属性指定其它名称），通过设置type属性来支持byType。

@Value

用在字段、方法、方法参数、注解属性中，用于设置默认值。

支持SpEL(Spring Expression Language)

@Value("${server.port}")
private int serverPort;

@Import

用在spring组件类、配置类上，可以用于注册一个普通的Java类对象到IoC容器中，但是常用的场景是自定义Stater时，用于导入外部配置类（如果配置类在标准的SpringBoot包结构下，会自动生效，无须导入）。

@ImportResource

用在spring组件类、配置类上，用于导入基于xml的spring配置文件。

@Conditional

条件化配置，满足一定的条件时才会生效。用在配置类或注册Bean的方法上。

SpringBoot提供了以下几种默认实现的条件化配置注解：

  • @ConditionalOnBean
  • @ConditionalOnMissingBean
  • @ConditionalOnClass
  • @ConditionalOnMissingClass
  • @ConditionalOnProperty
  • @ConditionalOnResource
  • @ConditionalOnWebApplication
  • @ConditionalOnJava
  • @ConditionalOnJndi
  • @Profile，激活指定的profile配置文件时生效。

自定义条件化配置注解时，需要先定义一个Condition接口的实现类作为注解属性值。

@ConfigurationProperties

配置属性，用在类上，可以自动将application.yml或application.properties中的配置项映射为类属性。

可以引入spring-boot-configuration-processor依赖，这样可以让ide对自定义配置项具有自动提示的能力

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-configuration-processor</artifactId>
    <optional>true</optional>
</dependency>
