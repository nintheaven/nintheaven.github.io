hugo搭建静态站点

建站 2021-03-01

hugo中文文档：https://www.gohugo.org/

Hugo In Action在线图书：https://livebook.manning.com/book/hugo-in-action/

入门

下载安装

https://github.com/spf13/hugo/releases

最简单的方式是，只需要下载对应操作系统的2进制文件，将其配置到PATH环境变量即可。

生成站点

hugo new site E:\blog

创建内容页

hugo new about.md

新建的页面位于content目录下。

hugo new posts/hello.md

使用同样的命令，在content/posts目录下，添加了一篇文章。

使用主题皮肤

https://www.gohugo.org/theme/

将喜欢的主题，例如ananke，通过git下载到themes目录下，文件夹名称与主题名称保持一致。

git clone git@github.com:theNewDynamic/gohugo-theme-ananke.git themes/ananke

运行hugo本地服务器

hugo server --theme=ananke --buildDrafts

访问 http://localhost:1313/ 即可看到一个静态站点成功创建了。

基础配置

hugo支持的配置文件格式依次是toml,yaml,json。这里使用yaml进行配置。

修改config.toml为config.yml

构建

修改content目录及其子目录下的文章，将文章头部的draft修改为false；

执行以下命令，将在public目录中，生成网站内容

hugo -D

部署

部署到github

在github上创建一个公开的仓库，名称为 lypzzz.github.io ，lypzzz为github用户名；

将public目录下的文件，全部推送到github仓库

cd public

git init

git remote add github git@github.com:lypzzz/lypzzz.github.io.git

git add --all

git commit -m "deploy"

git push -u github main

访问 https://lypzzz.github.io/ 即可看到成功部署站点到了github pages。

部署到gitee

gitee仓库名与github仓库名有点不一样，只需要使用用户名即可。

将gitee仓库添加为本地git仓库的远程仓库，然后push

git remote add gitee git@gitee.com:lypzzzzz/lypzzzzz.git

git push -u gitee main

然后，需要到gitee pages中，启动服务。之后有更新后，也需要去gitee pages中进行更新操作。

hugo常用命令

查看命令帮助

hugo help

查看配置

hugo config

启动hugo服务器

hugo server

构建内容

hugo -D
