软件架构

单体架构

SOA(Service Oriented Architecture, 面向服务的架构)

微服务(Micro Service)

服务切分

水平切分：同一应用的多个副本，部署到多台机器上。

垂直切分：按照业务，将一个应用拆分为多个子模块。

混合切分：将一个大的应用，按照业务功能垂直切分为多个子模块；对每一个子模块水平切分，进行集群部署。

分布式系统的设计原则

CAP原则

C（Consistency）一致性，数据一致性

A（Available）可用性，保证性能

P（Partition Tolerance），分区容错性，可扩展性。

不可能同时满足CAP原则中的3个目标。

BASE原则

BA（Basically Available）基本可用

S（Soft state）软状态，中间状态

E（Enventual consitency）最终一致性

BASE原则追求的是弱一致性，高可用性。
