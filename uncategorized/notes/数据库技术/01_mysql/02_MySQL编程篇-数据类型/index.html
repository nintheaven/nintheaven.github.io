MySQL编程篇-数据类型

: 未分类
发布于 : 2022-10-11

标准SQL（ANSI SQL）中规定了几种数据类型：数字、字符串、日期与时间。MySQL实现了标准中规定的数据类型。

数字

整数类型，可以使用 UNSIGNED 关键字指定为无符号整数。

整数 字节数 最小值(有符号) 最小值(无符号) 最大值(有符号) 最大值(无符号)
TINYINT 1 -128 0 127 255
SMALLINT 2 -32768 0 32767 65535
MEDIUMINT 3 -8388608 0 8388607 16777215
INTEGER 或 INT 4 -2147483648 0 2147483647 4294967295
BIGINT 8 -263 0 263-1 264-1

浮点数

注意：建议使用SQL标准推荐的 FLOAT(2) 形式指定浮点数精度，而不要使用MySQL支持的 FLOAT(5,2)的形式。

浮点数 字节数 说明 示例
FLOAT 4 单精度浮点数 FLOAT(2)，FLOAT(5,2)
DOUBLE 8 双精度浮点数 DOUBLE(2), DOUBLE(5,2)

定点数

MySQL中，使用 DECIMAL 或 NUMERIC 存储定点数。它们的作用一样。

salary DECIMAL(5,2)

字符串

CHAR 和 VARCHAR

CHAR 存储定长字符串，可以存储0-255个字符。实际长度小于最大长度的部分，用空格补齐。

VARCHAR 存储可变长度字符串，可以存储0-65535个字符。

BINARY 和 VARBINARY

与CHAR 和 VARCHAR 类似，不同的是存储2进制字符串，不支持使用字符集编码。

TEXT 和 BLOB

TEXT 存储长文本字符串，包括TINYTEXT, TEXT, MEDIUMTEXT, 和LONGTEXT 4种类型。

BLOG 以2进制的形式存储长文本字符串，包括 TINYBLOB, BLOB, MEDIUMBLOB, 和LONGBLOB 4种类型。

ENUM 和 SET

ENUM 枚举类型，可以在定义表的时候，为列设置预定义的一组字符串数据，该列的值必须是其中之一（或NULL）。

CREATE TABLE shirts (
    name VARCHAR(40),
    size ENUM('x-small', 'small', 'medium', 'large', 'x-large')
);

INSERT INTO shirts (name, size) VALUES ('dress shirt','large'), ('t-shirt','medium'),
  ('polo shirt','small');

SET 集合类型，可以在定义数据表时，为列设置预定义的一组字符串数据，该列的值必须是其中的一个或多个（或NULL）。

CREATE TABLE myset (col SET('a', 'b', 'c', 'd'));

INSERT INTO myset (col) VALUES 
	('a,d'), ('d,a'), ('a,d,a'), ('a,d,d'), ('d,a,d');
