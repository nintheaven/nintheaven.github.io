vue-cli入门

Vue技术栈 2022-10-11

https://cli.vuejs.org/zh/

安装

全局安装vue-cli

npm install -g @vue/cli

npm install -g @vue/cli-service-global

升级

npm update -g @vue/cli

常用命令

全部可用命令

创建Vue项目

  • 在新文件夹下创建项目

    vue create <app-name>

  • 在当前文件夹下创建项目

    vue create .

  • 使用ui界面创建项目

    vue ui

编译,启动服务

npm run serve

构建

npm run build

配置代理服务器

在项目根路径下创建文件 vue.config.js ，可以配置用于开发期间的代理服务器

部署vue项目到github pages

参考vue-cli文档：https://cli.vuejs.org/zh/guide/deployment.html#github-pages

准备一个github仓库

仓库名为 username.github.io，必须是公开的。

github pages访问地址为 https://username.github.io/

准备一个vue项目

mkdir blog

cd blog

vue create .

npm run build

在 vue.config.js 中设置 publicPath

因为仓库名为 username.github.io ，项目将被部署到 https://username.github.io/ ，这里 publicPath 默认为 /。否则，为仓库名

创建 deploy.sh 部署脚本

执行部署脚本

sh .\deploy.sh

确认部署结果

到github仓库的pages页面，看分支是否为master，如果不是，手动切换。

现在，访问 http://username.github.io 或者自定义的域名即可看到vue项目页面。

更新迭代vue项目

每次需要重新部署时，只需要运行deploy.sh部署脚本即可。

推送项目源代码到github仓库

github pages仓库中保存了build生成的静态文件，如果想将本地的vue项目代码也推送到github，可以进行以下操作。

创建github仓库（私有仓库，毕竟可能有私人信息），名称与vue项目同名。然后，推送vue代码到这个仓库

git add --all

git commit -m "commit"

git remote add origin git@github.com:jifeet/blog.git

git push -u origin master

报错

vue create hello-world 创建项目报错：

解决：

  • 清除npm缓存：npm cache clean --force

  • 修改 ~/.vuerc 文件，设置 useTaobaoRegistry = true
